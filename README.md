## 🎯 My Approach

### 1. Defect Identification
I identified 10 realistic defects by re-analysing the five modules of the SRS using the test cases from Assignment 2 as a reference. Defects were identified across functional, security, boundary, and integration testing dimensions.

### 2. Bug Report Document
Each of the 10 bug reports follows a professional format including: Bug ID, Module, Title, Description, Steps to Reproduce, Expected Result, Actual Result, Severity, Priority, Status, and Environment. Critical and security-related defects (BUG-001, BUG-002, BUG-007) were prioritised for immediate attention.

### 3. Bug Lifecycle Document
The lifecycle document covers all 9 possible defect statuses: New, Open, In Progress, Fixed, Retest, Closed, Reopened, Rejected, and Deferred. Each stage includes: a description, the Tester's role, the Developer's role, and valid status transitions. A visual flow diagram illustrates the full lifecycle path.

### 4. Defect Tracking Excel Sheet
The Excel workbook contains three tabs:
- **Defect Tracking Sheet** — All 10 defects with full tracking columns
- **Summary Dashboard** — Counts by module, status, and severity with Excel formulas
- **Bug Lifecycle Reference** — Quick reference table for all lifecycle stages
